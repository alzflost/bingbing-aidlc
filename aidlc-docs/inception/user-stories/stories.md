# User Stories

> 원본 기획서(family-profile-co-pilot-idea.md §5, §7) 기반.
> 5개 페르소나(아빠, 엄마, 아이(8세), 할머니, 게스트) 대상.
> 7시간 타임박스 내 데모 가능한 범위로 한정.
> 데모 핵심 카드: S3(같은 말, 다른 응답) + S4(가드레일)

스토리 분류: 시나리오 기반 (S1~S7 + 시스템 스토리)
우선순위: P0(데모 필수) / P1(데모 보강) / P2(시간 여유 시)
Acceptance Criteria: Given/When/Then 표준
비기능 요구사항: 응답 지연 3초 이내, 화자 식별 정확도 90%+ 목표

---

## S3: 같은 말, 다른 추천 [데모 핵심 카드 #1]

### US-S3-01: 화자별 개인화 응답 [P0]

**As a** 등록된 탑승자
**I want** 같은 말을 해도 내 선호도에 맞는 추천을 받길
**So that** 매번 취향을 설명하지 않아도 된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 엄마(비건 선호 LTM 저장됨)가 조수석 채널에서 인식됨 | "배고프다" 발화 | AgentCore Memory에서 actor_mother 프로파일 조회 → 비건 식당 추천 (3초 이내) |
| 2 | 아빠(고기 선호 LTM 저장됨)가 운전석 채널에서 인식됨 | "배고프다" 발화 | AgentCore Memory에서 actor_father 프로파일 조회 → 고기집 추천 (3초 이내) |
| 3 | 게스트(메모리 없음) | "배고프다" 발화 | 메모리 조회 스킵 → 일반 인기 식당 추천 (개인화 없음) |
| 4 | AgentCore Memory 조회 실패 (타임아웃/에러) | 등록 화자 발화 | 3초 타임아웃 후 일반 추천 fallback + 에러 로깅 |
| 5 | 아이(공룡/과학 관심 LTM) | "심심해" 발화 | 아이 LTM 기반 공룡 퀴즈/과학 콘텐츠 추천 |

## S4: 어린이 안전 모드 [데모 핵심 카드 #2]

### US-S4-01: 아이 차량 제어 차단 [P0]

**As a** 부모 (아빠/엄마)
**I want** 아이가 위험한 차량 제어를 요청해도 차단되길
**So that** 주행 중 안전이 보장된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 아이 페르소나(actor_child) + 고속도로 주행 중 | "창문 열어줘" 발화 | CEDAR Policy 평가 → 차단(고속 주행 중 어린이 차창 제어 금지) → 페르소나 맞춤 거절 ("OO야, 지금 고속도로라 위험해. 도착하면 열어줄게!") |
| 2 | 아이 페르소나 | "엄마한테 전화 걸어줘" 발화 | CEDAR Policy 평가 → 차단(어린이 BT전화 금지) → 우회 제안 ("대신 재미있는 퀴즈 할까?") |
| 3 | 아이 페르소나 | "랩 노래 시끄럽게 틀어줘" 발화 | 연령 필터(8세) 적용 → 연령 적합 콘텐츠 선별 + 볼륨 안전 한계(최대 60%) 적용 → 다른 음악 추천 |
| 4 | 아이 페르소나 + 정차 중 | "창문 열어줘" 발화 | CEDAR Policy 평가 → 조건부 허용(정차 중) → 부모 확인 요청 ("아빠, OO가 창문 열고 싶대요. 열어줄까요?") |
| 5 | 아이 페르소나 | "목적지 바꿔줘" 발화 | CEDAR Policy 평가 → 차단(어린이 내비 변경 금지) → 맞춤 거절 ("그건 아빠한테 부탁해봐!") |

## S1: 시동 ON — 화자별 차등 환영

### US-S1-01: 좌석 채널 기반 화자 인식 및 환영 [P0]

**As a** 탑승자 (모든 페르소나)
**I want** 시동이 켜지면 좌석 채널 기반으로 나를 인식하고 맞춤 환영을 해주길
**So that** 별도 조작 없이 바로 개인화된 서비스를 받을 수 있다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 아빠가 운전석에 탑승 (운전석 채널 = actor_father 매핑) | 시동 ON | 운전석 채널로 actor_father 식별 → AgentCore Memory에서 LTM 조회 → 개인화 환영 ("오늘 장모님 댁 막힘 없습니다. 안성휴게소 경유할까요?") (3초 이내) |
| 2 | 뒷좌석에 아이 탑승 (뒷좌석 채널) | 시동 ON + 아이 첫 발화 감지 | 뒷좌석 채널 + 어휘/말투 휴리스틱으로 actor_child 식별 → 연령 맞춤 환영 ("OO야, 가는 길에 공룡 퀴즈 할까?") |
| 3 | 매핑되지 않은 채널에서 음성 감지 | 시동 ON + 미등록 음성 | 자기소개 요청 프롬프트 ("안녕하세요! 이름을 말씀해주시겠어요?") |
| 4 | 자기소개 요청에 3초 내 응답 없음 | 타임아웃 | 게스트(actor_guest)로 자동 처리 + 기본 환영 ("안녕하세요, 무엇을 도와드릴까요?") |
| 5 | 화자 인식 처리 지연 3초 초과 | 시스템 지연 | 기본 환영 즉시 제공 + 백그라운드에서 인식 계속 → 인식 완료 시 개인화 전환 |

## S2: 동시 발화 충돌 해소

### US-S2-01: 운전자 우선 동시 발화 처리 [P0]

**As a** 운전자
**I want** 여러 사람이 동시에 말해도 안전 관련 요청이 우선 처리되길
**So that** 주행 중 안전이 지연되지 않는다

> 동시 발화 정의: 500ms 이내 겹치는 발화를 "동시"로 판정

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 고속도로 주행 중 | 아이 "만화 틀어줘" + 아빠 "경로 안내해줘" (500ms 이내 동시 발화) | CEDAR Policy: 주행 중 운전자 안전/내비 요청 절대 우선 → 아빠 요청 즉시 처리 → 아이 요청 큐잉 → 아빠 응답 완료 후 아이에게 "잠깐만, OO야! 만화 찾아볼게~" |
| 2 | 정차 중 | 엄마 "식당 찾아줘" + 아이 "노래 틀어줘" (500ms 이내 동시 발화) | 정차 중 = 운전자 특별 우선권 없음 → 선착순 처리 |
| 3 | 주행 중 | 할머니 "에어컨 줄여줘" + 아빠 "볼륨 올려줘" (500ms 이내 동시 발화) | 둘 다 비안전 요청 → 운전자 약간 우선 → 아빠 먼저 처리 후 할머니 순차 처리 |
| 4 | 큐잉된 요청 존재 | 우선 요청 처리 완료 | 큐잉된 요청 자동 처리 + 해당 화자 페르소나 톤으로 응답 (3초 이내) |

## S5: 노약자 응답 단순화

### US-S5-01: 할머니 맞춤 응답 [P1]

**As a** 할머니
**I want** 간단하고 천천히 답변을 받길
**So that** 쉽게 이해하고 사용할 수 있다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 할머니 페르소나(actor_elder) | "에어컨 좀 줄여줘" 발화 | 단순 핵심 응답 1문장 ("에어컨 온도 올렸습니다") + TTS 발화 속도 0.8x |
| 2 | 할머니 페르소나 | "OO한테 전화 좀 해줘" 발화 | 1회 재확인 ("OO님께 전화 걸까요?") → 확인 응답 후 실행 |
| 3 | 할머니 페르소나 | "오늘 날씨 어때?" 발화 | 1문장 핵심 답변 ("오늘 서울 맑고 25도입니다") + 추가 정보는 요청 시에만 |
| 4 | 할머니 페르소나 | 내비게이션 목적지 변경 시도 | CEDAR Policy 차단 → 정중한 거절 ("죄송합니다, 목적지 변경은 운전자분께 부탁드릴게요") |

## S6: 운행 후 화자별 메모리 누적

### US-S6-01: 트립 종료 시 화자별 학습 [P1]

**As a** 등록된 탑승자
**I want** 운행 중 내 선호가 자동으로 학습되길
**So that** 다음 탑승 시 더 나은 개인화를 받는다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 트립 중 엄마가 비건 식당을 선택함 (STM에 기록) | 시동 OFF | Reflection agent 실행 → actor_mother의 STM에서 "비건 식당 선택" 이벤트 추출 → LTM preference에 "비건 선호" confidence +0.1 증가 (기존 값 기반 점진적 강화) |
| 2 | 트립 중 아이가 공룡 퀴즈에 높은 반응 (STM에 기록) | 시동 OFF | Reflection agent 실행 → actor_child의 STM에서 "공룡 퀴즈 반응" 추출 → LTM preference에 "공룡 콘텐츠" 선호 저장 |
| 3 | 게스트 발화 존재 (STM에 기록) | 시동 OFF | 게스트 STM 데이터 전량 삭제, LTM 미적재 |
| 4 | Reflection agent 실행 실패 (타임아웃/에러) | 시동 OFF | 에러 로깅 + STM 데이터 보존 (삭제하지 않음) → 다음 시동 시 재시도 |
| 5 | 기존 LTM과 새 에피소드 충돌 (예: 기존 "고기 선호" vs 이번 트립 "비건 선택") | Reflection 실행 | 최신 이벤트 우선 반영하되 기존 preference 완전 삭제하지 않음 (confidence 조정만) |

## S7: 미등록 화자 처리

### US-S7-01: 게스트 프라이버시 보호 [P1]

**As a** 게스트 탑승자
**I want** 내 발화가 저장되지 않길
**So that** 프라이버시가 보호된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 3단계 매핑 모두 실패 (좌석 채널 미매핑 + 자기소개 미응답 + 휴리스틱 미해당) | 발화 | 게스트(actor_guest) 모드 자동 처리, 정보 조회만 허용 |
| 2 | 게스트 페르소나 | 차량 제어 요청 ("에어컨 켜줘") | CEDAR Policy 차단 → 중립 톤 거절 ("죄송합니다, 차량 제어는 등록된 탑승자만 가능합니다") |
| 3 | 게스트 페르소나 | "오늘 날씨 어때?" | 정상 응답 (정보 조회 허용, 3초 이내) |
| 4 | 게스트 발화 이력 존재 | 트립 종료 (시동 OFF) | STM에서 게스트 발화 데이터 전량 삭제, LTM 미적재 |

## 시스템 스토리 (Cross-cutting)

### US-SYS-01: 화자 매핑 상태 머신 [P0]

**As a** 시스템
**I want** 3단계 매핑 전략(좌석 채널 → 자기소개 → 휴리스틱)으로 화자를 식별하길
**So that** 모든 시나리오에서 올바른 페르소나가 적용된다

> 매핑 전략:
> 1. Primary — 좌석 채널 기반 (운전석 마이크 = 주 운전자 고정)
> 2. Secondary — 트립 시작 1회 자기소개 (spk_N ↔ 페르소나 바인딩)
> 3. Tertiary — 어휘/말투 콘텐츠 기반 휴리스틱 (어린이/노약자 보정)

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | Idle 상태 | 시동 ON | Onboarding 상태 전환 → 좌석 점유 센서 확인 → 좌석 채널 기반 1차 매핑 시도 |
| 2 | Onboarding 상태 + 좌석 채널 매핑 성공 | 등록된 채널에서 발화 | Active 상태 전환, spk_N ↔ actor_id 바인딩 확정 |
| 3 | Onboarding 상태 + 미매핑 채널 발화 | 자기소개 요청 후 응답 수신 | 자기소개 내용으로 페르소나 바인딩 → Active 상태 전환 |
| 4 | Active 상태 + 신규 미등록 음성 | 3단계 매핑 모두 실패 | Guest(actor_guest) 처리, 메모리 미적용 |
| 5 | Active 상태 | 시동 OFF | Trip-end 상태 → Reflection agent 실행 → 완료 후 Idle 복귀 |
| 6 | 상태 전환 중 예외 발생 | 시스템 에러 | 이전 상태 유지 (fail-safe) + 에러 로깅 |

### US-SYS-02: 평가 지표 실측 [P2]

**As a** 시스템 운영자
**I want** 핵심 품질 지표를 측정하길
**So that** 시스템 품질을 모니터링할 수 있다

> 핵심 지표 3종 (원본 Ops-01 기반):
> - 화자 식별 정확도 (mapping accuracy): 목표 90%+
> - 페르소나 일관성 (correct persona response rate): 목표 95%+
> - 가드레일 누락률 (policy violation rate): 목표 0%

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 화자 발화 처리 완료 | 매 응답 | 화자 식별 정확도 지표 업데이트 (AgentCore Evaluations) |
| 2 | 페르소나 맞춤 응답 생성 | 매 응답 | 페르소나 일관성 지표 업데이트 |
| 3 | CEDAR Policy 평가 실행 | 도구 호출 시도 시 | 가드레일 누락률 지표 업데이트 (차단해야 할 것을 통과시킨 비율) |
| 4 | 대시보드 조회 | 운영자 요청 | 3개 지표 + 트립별 추이 표시 |
