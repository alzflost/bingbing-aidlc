# User Stories

스토리 분류: 시나리오 기반 (S1~S7 + 시스템 스토리)
우선순위: P0/P1/P2
Acceptance Criteria: Given/When/Then 표준 (3~5개, 정상/예외 포함)
비기능 요구사항: 기능 스토리의 AC에 포함

---

## S1: 시동 ON — 화자별 차등 환영

### US-S1-01: 자동 화자 인식 및 환영 [P0]

**As a** 탑승자 (모든 페르소나)
**I want** 시동이 켜지면 시스템이 나를 자동으로 인식하고 맞춤 환영을 해주길
**So that** 별도 조작 없이 바로 개인화된 서비스를 받을 수 있다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 이전 트립에서 학습된 아빠가 운전석에 탑승 | 시동 ON | 자동 인식 후 개인화 환영 ("오늘 장모님 댁 막힘 없습니다") |
| 2 | 이전 트립에서 학습된 유아가 뒷좌석에 탑승 | 시동 ON + 유아 첫 발화 감지 | 자동 인식 후 연령 맞춤 환영 ("OO야, 가는 길에 공룡 퀴즈 할까?") |
| 3 | 처음 탑승하는 화자 | 시동 ON + 미등록 음성 감지 | 자연스러운 대화로 확인 시도 ("안녕하세요! 혹시 누구신가요?") |
| 4 | 확인 시도에 응답 없는 화자 | 3초 내 응답 없음 | 게스트로 자동 처리, 기본 환영 |
| 5 | 시스템 응답 지연 3초 초과 | 화자 인식 처리 중 | 타임아웃 후 기본 환영 제공, 백그라운드에서 인식 계속 |

## S3: 같은 말, 다른 추천

### US-S3-01: 화자별 개인화 응답 [P0]

**As a** 등록된 탑승자
**I want** 같은 말을 해도 내 선호도에 맞는 추천을 받길
**So that** 매번 취향을 설명하지 않아도 된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 엄마(비건 선호 LTM 저장됨) | "배고프다" 발화 | 비건 식당 추천 |
| 2 | 아빠(고기 선호 LTM 저장됨) | "배고프다" 발화 | 고기집 추천 |
| 3 | 게스트(메모리 없음) | "배고프다" 발화 | 일반 인기 식당 추천 (개인화 없음) |
| 4 | 청소년(카페 선호 LTM) | "배고프다" 발화 | 카페/디저트 추천 |
| 5 | AgentCore Memory 조회 실패 | 화자 발화 | 일반 추천 fallback + 에러 로깅 |

## S4: 어린이 안전 모드

### US-S4-01: 유아 차량 제어 차단 [P0]

**As a** 부모 (아빠/엄마)
**I want** 유아가 위험한 차량 제어를 요청해도 차단되길
**So that** 주행 중 안전이 보장된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 유아 페르소나 + 고속도로 주행 중 | "창문 열어줘" 발화 | Policy 차단 + 맞춤 거절 ("OO야, 지금 고속도로라 위험해. 도착하면 열어줄게!") |
| 2 | 유아 페르소나 | "전화 걸어줘" 발화 | Policy 차단 + 우회 제안 ("대신 재미있는 퀴즈 할까?") |
| 3 | 유아 페르소나 | "시끄러운 음악 틀어줘" 발화 | 연령 적합 콘텐츠 + 볼륨 안전 한계 적용 |
| 4 | 유아 페르소나 + 정차 중 | "창문 열어줘" 발화 | 부분 허용 (창문 일부만 개방) 또는 부모 확인 요청 |

### US-S4-02: 청소년 콘텐츠 가드레일 [P0]

**As a** 부모 (아빠/엄마)
**I want** 청소년이 부적절한 콘텐츠에 접근하지 못하게 하길
**So that** 안전한 차량 환경이 유지된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 청소년 페르소나 | 성인 콘텐츠 검색 시도 | 콘텐츠 필터 차단 + 대안 제시 |
| 2 | 청소년 페르소나 | 차량 제어 요청 | Policy 차단 + 캐주얼 톤 거절 ("그건 아빠한테 부탁해봐") |
| 3 | 청소년 페르소나 | 연령 적합 음악 요청 | 정상 실행 |
| 4 | 청소년 페르소나 | 웹 검색 (일반) | 정상 실행 + 콘텐츠 필터 적용 |

## S2: 동시 발화 충돌 해소

### US-S2-01: 컨텍스트 기반 동시 발화 처리 [P1]

**As a** 운전자
**I want** 여러 사람이 동시에 말해도 상황에 맞게 우선 처리되길
**So that** 안전 관련 요청이 지연되지 않는다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 고속도로 주행 중 | 유아 "만화 틀어줘" + 아빠 "경로 안내해줘" 동시 발화 | 아빠(안전/내비) 우선 처리, 유아 요청 큐잉 후 순차 처리 |
| 2 | 정차 중 | 엄마 "식당 찾아줘" + 청소년 "음악 바꿔줘" 동시 발화 | 선착순 처리 (운전자 특별 우선권 없음) |
| 3 | 주행 중 | 어르신 "에어컨 줄여줘" + 아빠 "볼륨 올려줘" 동시 발화 | 둘 다 비안전 → 운전자 약간 우선 |
| 4 | 큐잉된 요청 존재 | 우선 요청 처리 완료 | 큐잉된 요청 자동 처리 + 해당 화자에게 응답 |

## S5: 노약자 응답 단순화

### US-S5-01: 어르신 맞춤 응답 [P1]

**As a** 어르신
**I want** 간단하고 천천히 답변을 받길
**So that** 쉽게 이해하고 사용할 수 있다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 어르신 페르소나 | "에어컨 줄여줘" 발화 | 단순 핵심 응답 ("에어컨 온도 올렸습니다") + 느린 발화 속도 |
| 2 | 어르신 페르소나 | 전화 발신 요청 | 1회 재확인 ("OO님께 전화 걸까요?") 후 실행 |
| 3 | 어르신 페르소나 | 복잡한 질문 | 1문장 핵심 답변 우선, 추가 정보는 요청 시에만 |

## S6: 운행 후 화자별 메모리 누적

### US-S6-01: 트립 종료 시 학습 [P1]

**As a** 등록된 탑승자
**I want** 운행 중 내 선호가 자동으로 학습되길
**So that** 다음 탑승 시 더 나은 개인화를 받는다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 트립 중 엄마가 비건 식당 선택 | 시동 OFF | Reflection agent가 엄마 LTM에 비건 선호 강화 저장 |
| 2 | 트립 중 유아가 공룡 퀴즈 반응 | 시동 OFF | 유아 LTM에 공룡 콘텐츠 선호 저장 |
| 3 | 게스트 발화 존재 | 시동 OFF | 게스트 발화 미저장, STM에서 삭제 |
| 4 | Reflection agent 실행 실패 | 시동 OFF | 에러 로깅 + STM 데이터 보존 (다음 시동 시 재시도) |

## S7: 미등록 화자 처리

### US-S7-01: 게스트 프라이버시 보호 [P0]

**As a** 게스트 탑승자
**I want** 내 발화가 저장되지 않길
**So that** 프라이버시가 보호된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 미등록 음성 감지 + 매핑 실패 | 발화 | 게스트 모드 처리, 정보 조회만 허용 |
| 2 | 게스트 페르소나 | 차량 제어 요청 | 차단 + 중립 톤 거절 |
| 3 | 게스트 페르소나 | "오늘 날씨 어때?" | 정상 응답 (정보 조회 허용) |
| 4 | 게스트 발화 이력 | 트립 종료 | 모든 발화 데이터 삭제, LTM 미적재 |

## 시스템 스토리 (Cross-cutting)

### US-SYS-01: 화자 매핑 상태 머신 [P0]

**As a** 시스템
**I want** 화자 매핑 상태를 일관되게 관리하길
**So that** 모든 시나리오에서 올바른 페르소나가 적용된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | Idle 상태 | 시동 ON | Onboarding 상태 전환, 좌석 점유 확인 |
| 2 | Onboarding 상태 | 화자 인식 완료 | Active 상태 전환, spk_N ↔ actor_id 바인딩 유지 |
| 3 | Active 상태 | 신규 미등록 음성 | 자연스러운 대화 확인 시도, 실패 시 Guest 처리 |
| 4 | Active 상태 | 시동 OFF | Trip-end 상태 → Reflection → Idle |
| 5 | 상태 전환 실패 | 예외 발생 | 이전 상태 유지 + 에러 로깅 (fail-safe) |

### US-SYS-02: 동적 페르소나 등록 [P1]

**As a** 시스템
**I want** 대화를 통해 새 페르소나를 자동 생성하고 기존 페르소나를 보강하길
**So that** 사용할수록 인식 정확도가 향상된다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 새 탑승자 + 대화 중 이름/나이 언급 | AI 분석 | 기본 프로파일 자동 생성 (이름, 추정 연령대, 기본 권한) |
| 2 | 기존 페르소나 + 새로운 선호 표현 | 대화 패턴 분석 | 페르소나 속성 점진적 업데이트 |
| 3 | 자동 생성된 프로파일 | 다음 트립 탑승 | 학습된 프로파일로 자동 인식 |

### US-SYS-03: 평가 지표 실측 [P1]

**As a** 시스템 운영자
**I want** 화자 식별 정확도, 페르소나 일관성, 가드레일 누락률을 실시간 측정하길
**So that** 시스템 품질을 모니터링하고 개선할 수 있다

**Acceptance Criteria:**

| # | Given | When | Then |
|---|---|---|---|
| 1 | 화자 발화 처리 완료 | 매 응답 | 화자 식별 정확도 지표 업데이트 |
| 2 | 페르소나 맞춤 응답 생성 | 매 응답 | 페르소나 일관성 지표 업데이트 |
| 3 | Policy 평가 실행 | 도구 호출 시도 | 가드레일 누락률 지표 업데이트 |
| 4 | 대시보드 조회 | 사용자 요청 | 3개 지표 실시간 표시 |
