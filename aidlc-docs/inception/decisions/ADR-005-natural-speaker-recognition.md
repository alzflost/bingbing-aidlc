## Context and Problem Statement

차량 탑승 시 화자(speaker) 식별 → 페르소나 매핑은 본 시스템의 시작점이다. 매번 강제 자기소개를 요구하면 UX가 크게 손상되며, 반대로 인식 실패를 방치하면 잘못된 페르소나로 응답해 안전 / 개인화 실패를 초래한다.

초기 설계는 "트립 시작 시 1회 자기소개를 강제"였으나, 사용자 피드백("대화형으로 자연스럽게 풀거나 옵셔널하게 하는건 어때? 너무 어색할꺼같아 매번 자기소개하면서 타면" — 2026-05-20T09:20:00Z) 이후 자연스러운 인식 전략으로 재설계.

본 결정은 화자 인식의 단계 / 트리거 / fallback을 어떻게 설계할지 결정한다.

## Considered Options

1. **강제 자기소개 (초기안)** — 시동 ON 시 모든 탑승자에게 "이름을 말씀해주세요" 요청. UX 손상 큼
2. **Voiceprint 등록 시스템** — 사전 등록된 음성 지문과 매칭. 등록 절차 + 모델 학습 필요. 7시간 외 스코프
3. **3-step Natural Recognition** — 좌석 채널 자동 매핑 → 자기소개 확인 (필요 시) → 어휘/말투 휴리스틱. 강제 프롬프트 제거

## Decision Outcome

**옵션 3 (3-step Natural Recognition)을 채택**한다.

### 단계

| 단계 | 트리거 | 동작 | 데이터 출처 |
|---|---|---|---|
| **1. 좌석 채널 자동 매핑 (Primary)** | 시동 ON | 운전석 채널 → driver=true 자동 부여, 조수석 채널 → adult passenger 추정 | 좌석 점유 센서, 채널 메타 |
| **2. 자기소개 바인딩 (Secondary)** | 미매핑 채널의 첫 발화 감지 | 자연스러운 확인 ("안녕하세요! 누구신지 알 수 있을까요?") | 사용자 발화 |
| **3. 어휘/말투 휴리스틱 (Tertiary)** | Step 1, 2 후 보정 | 어휘 / 말투로 age_group 추정 (child / elder 보정) | LLM 분류 (Claude Haiku 4.5) |

### Fallback

- Step 2 응답이 없거나 매핑 실패 → `relationship=guest` 자동 처리, 정보 조회만 허용 (US-S7-01)
- 이전 트립에서 학습된 가족은 Step 1 결과를 LTM과 매칭해 자동 인식 — 별도 자기소개 불필요 (FR-01.4 해당, ADR-003 LTM 활용)

### UX 원칙

- **강제 자기소개 프롬프트 제거**: 매번 묻지 않음
- **확신도가 낮을 때만 확인**: 추론 신뢰도 기준 임계값 미만에서만 자연 대화로 확인
- **명시적 확인은 옵셔널**: 사용자가 직접 "나야, 아빠야"라고 말하면 즉시 반영하되 강제하지 않음

## Consequences

- ✅ **자연스러운 UX**: 매번 자기소개를 요구하지 않아 운전 중 인지 부담 최소화
- ✅ **데모 임팩트**: "차에 타자마자 인사하면 된다"는 시연이 가능해 피치 카드로 강함
- ✅ **이전 트립 학습 활용**: ADR-003 LTM 자동 인식과 결합하면 등록된 가족은 무조건 별도 확인 불필요 (FR-01.4)
- ✅ **프라이버시 friendly**: 미등록 화자는 guest 처리되어 메모리 미저장 (FR-05.4)
- ⚠️ **휴리스틱 정확도 의존**: Step 3의 child / elder 추정은 LLM 분류 품질에 의존. 잘못된 분류 시 권한 오부여 가능 — Construction 단계 PBT invariant("어떤 입력에서도 child 권한 차단은 유지")로 안전 마진 확보
- ⚠️ **확신도 임계값 튜닝 필요**: 낮으면 매번 묻고, 높으면 오인식 — 데모 시나리오 기반 5건 정도 사전 튜닝 필요
- ⚠️ **데모 시 명시적 시연 필요**: "강제 자기소개 없음"은 안 보이는 디자인이라 데모에서 명시 (피치 0:30~1:30 슬롯)

## Prior Art

본 결정은 prior art 없는 **신규 UX 결정**이다. 사용자 검토 피드백에서 직접 도출되었으며, 본 프로젝트의 Human-in-the-Loop 증거 중 가장 강력한 사례 중 하나다.

비교 대상으로 검토한 다음 사례들은 본 결정과 다르다.

| 비교 대상 | 화자 인식 방식 | 본 결정과의 차이 |
|---|---|---|
| LG U+ ixi-drive | 단일 화자(주 운전자) 가정, 화자 분기 부재 | 다중 화자 분기 자체가 부재 |
| 일반 음성 비서 (Siri / Alexa) | Voiceprint 등록 (사전 enrollment 필요) | 강제 등록 절차 필요 |
| 본 프로젝트 ADR-005 | 3-step natural — 등록 절차 / 강제 자기소개 / Voiceprint 모두 없음 | 등록 없이 자연스럽게 인식, fallback은 guest |
